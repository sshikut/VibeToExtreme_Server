## 2. 핵심 기술 구현 성과 (Technical Highlights)

### 2.1 C++ 서버: IOCP 기반 고성능 아키텍처
- **비동기 I/O 모델:** `GetQueuedCompletionStatus`를 활용한 워커 스레드 풀 운영으로 컨텍스트 스위칭 비용 최소화.
- **세션 관리자(SessionManager):** 최대 10,000개의 세션을 사전에 풀링(Pooling)하여 런타임 할당 오버헤드 제거.
- **동시성 제어:** 브로드캐스팅 시 발생할 수 있는 락 경합을 방지하기 위해 `std::shared_mutex`(읽기-쓰기 락) 아키텍처 설계 반영.

### 2.2 Unity 클라이언트: Zero-Allocation 역직렬화
- **메모리 최적화:** `unsafe` 블록과 `fixed` 포인터를 사용하여 수신된 바이트 배열을 구조체로 직접 캐스팅. 가비지 컬렉터(GC)의 개입을 원천 차단하여 프레임 드랍 방지.
- **부하 분산:** `TCP_NODELAY` 옵션 활성화 및 패킷 조립(Stream Parsing) 로직 구현을 통해 패킷 뭉침 현상(Sticky Packet) 해결.

### 2.3 실시간 동기화 시스템
- **Handshake:** 로그인 시 서버로부터 고유 세션 ID를 발급받는 절차 구현.
- **Life-Cycle:** 유저 입장(Spawn), 이동(Move), 퇴장(Leave) 패킷의 서버 중계 및 클라이언트 렌더링 연동 완료.
- **Interpolation:** 선형 보간(`Vector3.Lerp`)을 적용하여 네트워크 레이턴시 상황에서도 부드러운 캐릭터 움직임 구현.

## 3. AI 페어 프로그래밍 및 트러블슈팅 사례

본 프로젝트는 AI와의 기술적 토론을 통해 단순 기능 구현을 넘어 **상용 서비스 레벨의 예외 상황(Edge Case)**을 검증하는 방식으로 진행되었습니다.

### 💡 사례 1: 구조체 메모리 오프셋(Offset) 불일치 해결
- **문제:** 클라이언트 간 ID가 '0'으로 파싱되어 명부(Dictionary)에 유저가 등록되지 않는 치명적 버그 발생.
- **해결:** AI와 함께 C++과 C#의 메모리 패딩(Padding) 및 헤더 포함 여부에 따른 오프셋을 분석. `ptr + sizeof(PacketHeader)` 주소 연산의 논리적 결함을 찾아내어 수정 완료.

### 💡 사례 2: 패킷 폭풍(Packet Storm) 방어 설계
- **문제:** 1만 명 동시 접속 시 발생할 수 있는 `O(N^2)` 패킷 복사 및 전송 부하 예측.
- **해결:** 더미 클라이언트 제작 시 '지터 타이머(Jitter Timer)'를 도입하여 전송 시점을 분산시키고, 서버 단에서 시야 처리(Spatial Partitioning) 도입 필요성을 아키텍처 설계에 반영.
